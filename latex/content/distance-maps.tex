\subsection{Point cloud registration}\label{sec:distance-maps:background:registration}
One of the most popular approaches for point cloud registration is
the \glslink{icp}{\glsentryfull{icp}} algorithm.
Besl and McKay~\cite{besl-mckay:tpami:1992} address registration of 3D~shapes
represented as point clouds, introducing the standard \gls{icp} algorithm.
The following two steps can summarize this algorithm:
(i) compute correspondences between the source ($z_i$) and target ($p_i$)
cloud points (closest nearest neighbor based on Euclidean distance); and
(ii) estimate the rigid transformation of the source \gls{wrt} target cloud
($\mathbf{X}$) that minimizes a point-to-point distance measure between
corresponding points (see \eqref{eq:distance-maps:background:point-to-point}).
Chen and Medioni~\cite{chen-medioni:icra:1991} exploit the tendency of
most range data to be locally planar and introduce a point-to-plane variant
of \gls{icp}. The proposed method exploits surface normal information ($n_i$).
Instead of minimizing the difference between the projected source point and
the target point, the point-to-plane error metric minimizes
the squared dot product of the surface normal with
the difference between the expected projected source point and
the reference target point
(see \eqref{eq:distance-maps:background:point-to-plane}).
Indeed, Rusinkiewicz and Levoy~\cite{rusinkiewicz-levoy:im:2001} compared
\gls{icp} variants with respect to point selection,
data association, weighting of associated pairs, outlier rejection, and
error metrics. The point-to-plane~\cite{chen-medioni:icra:1991}
metric demonstrated significantly better performance than
the point-to-point~\cite{besl-mckay:tpami:1992} formulation in the experiments.
\begin{equation}\label{eq:distance-maps:background:point-to-point}
\mathbf{X} = \argmin_\mathbf{X} \sum_i \| \mathbf{X} z_i - p_i \|^2
\end{equation}
\begin{equation}\label{eq:distance-maps:background:point-to-plane}
\mathbf{X} = \argmin_\mathbf{X} \sum_i \| n_i^T \cdot
\left( \mathbf{X} z_i - p_i \right) \|^2
\end{equation}

Unlike traditional \gls{icp} methods, which rely on explicit point
correspondences, the \glslink{ndt}{\glsentryfull{ndt}}
proposed by Biber and Strasser~\cite{biber-strasser:iros:2003} performs
2D~point cloud registration by representing the environment
with local normal distributions. The 2D~space is discretized into cells.
Each cell is associated with a Gaussian distribution defined by
the mean and covariance of the cell points,
which provide information about the surface those points represent.
Then, Biber and Strasser~\cite{biber-strasser:iros:2003} formulate
2D~point cloud registration as the minimization of an error function
derived from these distributions, optimized using the Gauss--Newton algorithm.
Experimental results showed that \gls{ndt}~\cite{biber-strasser:iros:2003} was
able to map large areas without requiring odometry.
Magnusson~\etal~\cite{magnusson:jfr:2007} later extended \gls{ndt} to
3D~point cloud registration by subdividing space into voxels,
each modelled as a normal distribution estimated from
the points contained within it. The alignment is obtained by
evaluating the likelihood of points from one scan under
the distributions of the other scan.
Compared with point-to-point \gls{icp}~\cite{besl-mckay:tpami:1992} using
kd-tree nearest-neighbor search~\cite{bentley:cacm:1975},
3D-\gls{ndt}~\cite{magnusson:jfr:2007} achieves
lower computation cost by avoiding explicit correspondence search,
while also offering a more compact representation of 3D~point cloud
compared to raw data.

While leveraging
\gls{ndt}~\cite{biber-strasser:iros:2003,magnusson:jfr:2007}-like
representations of the scene, \gls{gicp}~\cite{segal:rss:2009} and
\gls{nicp}~\cite{serafin-grisetti:iros:2015} still retain the \gls{icp}
structure for cloud registration.
\gls{gicp}~\cite{segal:rss:2009} retains the standard \gls{icp}
structure for correspondence estimation, typically using Euclidean
closest-neighbor search accelerated with a kd-tree~\cite{bentley:cacm:1975}.
The main modification lies in the minimization objective function,
introducing a plane-to-plane error metric. \gls{gicp} models
local surface information around each point, estimated from
neighboring points via \gls{pca}. By accounting for the covariance of
the matched points, \gls{gicp} weights residuals based on
local geometric uncertainty. Experiments showed that
\gls{gicp}~\cite{segal:rss:2009} was more robust to incorrect correspondences
and more accurate than standard point-to-point and point-to-plane
\gls{icp}~\cite{besl-mckay:tpami:1992,chen-medioni:icra:1991}.
\gls{nicp}~\cite{serafin-grisetti:iros:2015} extends
\gls{gicp}~\cite{segal:rss:2009} not only in the optimization stage but also
during correspondence search.
Rather than relying purely on Euclidean nearest-neighbor matching,
\gls{nicp}~\cite{serafin-grisetti:iros:2015} uses a line-of-sight-based
association step and rejects candidate pairs when normals are not well defined,
when point distance exceeds a threshold, when curvature mismatch is too large,
or when the normals are insufficiently aligned. In the optimization step,
\gls{nicp}~\cite{serafin-grisetti:iros:2015} augments each point with
its normal and minimizes a Mahalanobis distance in this 6D space.
In the experiments, \gls{nicp}~\cite{serafin-grisetti:iros:2015} was more
accurate than \gls{gicp}~\cite{segal:rss:2009} and
\gls{ndt}~\cite{magnusson:jfr:2007}
registration algorithms on the evaluated datasets.

Furthermore, Montesano~\etal~\cite{montesano:iros:2005} present
a probabilistic scan matching method for 2D~point cloud registration.
The method~\cite{montesano:iros:2005} builds a set of possible correspondences
using an individual compatibility test based on the Mahalanobis distance
between pairs of points, while accounting for correspondence uncertainty.
Then, Montesano~\etal~\cite{montesano:iros:2005} maximize
the probability of scan alignment over the points correspondence distribution.
The algorithm~\cite{montesano:iros:2005} showed faster convergence
rates than standard \gls{icp}~\cite{besl-mckay:tpami:1992}.
Instead of selecting a single set of correspondences as
Montesano~\etal~\cite{montesano:iros:2005}, Censi~\cite{censi:icra:2006}
considers all possible correspondences and generates alignment hypotheses
through the \gls{ght}, while avoiding explicit point-to-point correspondences.
Censi~\cite{censi:icra:2006} computes alignment within
a probabilistic framework that incorporates a motion model and estimates
for registration uncertainty, accounting for surface normals.

Similarly, the GMapping~\cite{grisetti:tro:2007} and
Cartographer~\cite{hess:icra:2016} \gls{slam} algorithms use \gls{mle}
probabilistic formulations to estimate the scan alignment \gls{wrt} the map.
The scan matcher in GMapping~\cite{grisetti:tro:2007} seeks
the most likely robot pose by matching the current observation against
the map built so far. The \gls{mle} formulation estimates
the robot pose given the map, the current scan, and an initial pose guess.
This formulation performs a gradient-descent search over
the observation likelihood function defined on the grid map, based on the
beam-endpoint sensor model~\cite{thrun:book:2005}.
Cartographer~\cite{hess:icra:2016} optimizes the scan pose relative to
the current local submap using a Ceres~\cite{ceres:sw:2023}-based scan matcher.
This matcher estimates the pose that maximizes the occupancy probabilities of
the scan points projected onto the submap. The problem is formulated as
a non-linear least-squares optimization, using a smooth version of
the occupancy probability values associated with the projected scan points.

Precomputed spatial gradients may be leveraged for the 2D~point cloud
registration without requiring explicit point-to-point correspondences.
HectorSLAM~\cite{kohlbrecher:ssr:2011} uses spatial gradients over
the 2D~occupancy grid map ($M$). The scan matching finds the pose that maximizes
the occupancy probability of where the beam endpoints land on occupied cells
(see \eqref{eq:distance-maps:background:hectorslam}).
The method uses bilinear interpolation over the occupancy map and approximates
its spatial derivatives, applying then a Gauss-Newton update.
PerfectMatch~\cite{lauer:robocup:2006,sobreira:jint:2019} focuses on
a soccer field markings environment for robot soccer.
This method formulates the 2D~registration problem based on distance maps, using
the \gls{rprop}~\cite{riedmiller-braun:icnn:1993} gradient-descent algorithm.
Each point is evaluated by the distance map ($\mathcal{D}_M$)
that gives the distance to the closest field marking through a
robust M-estimator ($\rho$)
(see \eqref{eq:distance-maps:background:perfectmatch}).
The algorithm considers that the distance map is over the entire soccer field
environment, only considering scan-to-map alignment.
Although distance maps are continuous and thus differentiable,
the authors approximate the final estimation covariance based on the second
derivative of the error assuming an environment with orthogonal walls,
equivalent to the line markings of soccer fields.
Sobreira~\etal~\cite{sobreira:jint:2019} benchmarked in a simulation environment
PerfectMatch~\cite{lauer:robocup:2006} against
\gls{icp}~\cite{besl-mckay:tpami:1992} and
\gls{ndt}~\cite{biber-strasser:iros:2003}.
PerfectMatch~\cite{lauer:robocup:2006} was faster than both
\gls{icp}~\cite{besl-mckay:tpami:1992} and
\gls{ndt}~\cite{biber-strasser:iros:2003}, given that
PerfectMatch~\cite{lauer:robocup:2006} uses lookup tables
for the error function.
Additionally, the PerfectMatch~\cite{lauer:robocup:2006}
algorithm demonstrated a higher tolerance to orientation initialization errors
than \gls{icp}~\cite{besl-mckay:tpami:1992} and
\gls{ndt}~\cite{biber-strasser:iros:2003}.

\begin{equation}\label{eq:distance-maps:background:hectorslam}
\mathbf{X} = \argmin_\mathbf{X} \sum_i
\left[ 1 - M \left(\mathbf{X} z_i\right) \right]^2
\end{equation}
\begin{equation}\label{eq:distance-maps:background:perfectmatch}
\mathbf{X} = \argmin_\mathbf{X} \sum_i
\rho \left( \mathcal{D}_M \left( \mathbf{X} z_i \right) \right)
\end{equation}

Overall, most \gls{icp} methods rely on explicit point-to-point correspondences,
requiring two steps per optimization iteration: data association and
pairwise error computation between source and target points.
As for probabilistic-based approaches, these methods typically require solving a
\gls{map} problem and having a map estimate of the environment over time, which
is not compatible with scan-to-scan registration~\cite{segal:rss:2009}.
In contrast, distance maps
should be compatible with both scan-to-scan and scan-to-map alignment,
providing a nearest neighbor framework between source and target clouds.
While the PerfectMatch~\cite{lauer:robocup:2006} algorithm is specific
to scan-to-map registration, an adaptation of the original approach
should be compatible with scan-to-scan alignment.
Moreover, distance maps provide a precomputed lookup table
that directly allows evaluating the registration error of each source point
without requiring an explicit data association step.
This property allows faster computation during point cloud registration than
\gls{icp}~\cite{besl-mckay:tpami:1992} and
\gls{ndt}~\cite{biber-strasser:iros:2003}, as showed in
Sobreira~\etal~\cite{sobreira:jint:2019}.

\subsubsection{Problem formulation}\label{sec:distance-maps:background:least-squares:problem-formulation}

Least-squares minimization arises in robotics when estimating an unknown state
from noisy sensor measurements. Let $\mathbf{x}$ denote non-observable
state variables and $\mathbf{z}$ the measurements.
In the scope of this PhD~Thesis, the
measurements correspond to the perception of the environment obtained
by on-board sensors of a mobile robot, namely 2D~laser scanners.
The state is assumed to follow a prior distribution $p(\mathbf{x})$,
while the conditional distribution of the measurements given the state,
$p(\mathbf{z}|\mathbf{x})$, is assumed known,
commonly referred to as the observation model. Then, the goal
is to estimate the most probable state given the measurements,
\ie, the posterior $p(\mathbf{x}|\mathbf{z})$~\cite{grisetti:robotics:2020}.
This problem represents a \gls{map} estimation problem, as shown in
\eqref{eq:distance-maps:background:least-squares:map}.
\begin{equation}\label{eq:distance-maps:background:least-squares:map}
\mathbf{x}^* = \argmax_\mathbf{x} p(\mathbf{x}|\mathbf{z})
\end{equation}

By applying Bayes' rule, shown in
\eqref{eq:distance-maps:background:least-squares:bayes}, the posterior
distribution $p(\mathbf{x}|\mathbf{z})$ becomes proportional to
the likelihood of the measurements given the state
$p(\mathbf{z}|\mathbf{x})$ multiplied by the prior $p(\mathbf{x})$.
If a uniform prior over the admissible values of $\mathbf{x}$ is assumed,
then $p(\mathbf{x})$ is constant and the
posterior $p(\mathbf{x}|\mathbf{z})$ depends only on the measurement
distribution conditioned by the state $p(\mathbf{z}|\mathbf{x})$.
\begin{equation}\label{eq:distance-maps:background:least-squares:bayes}
p(\mathbf{x}|\mathbf{z}) =
\frac{p(\mathbf{z}|\mathbf{x})p(\mathbf{x})}{p(\mathbf{z})}
\propto p(\mathbf{z}|\mathbf{x})p(\mathbf{x})
\propto p(\mathbf{z}|\mathbf{x})
\end{equation}

Next, considering multiple measurements $\mathbf{z}^{[i]}$, these are assumed
conditionally independent given $\mathbf{x}$, resulting in
\eqref{eq:distance-maps:background:least-squares:independence}.
Under the assumption that each measurement follows a Gaussian distribution,
\eqref{eq:distance-maps:background:least-squares:measurement} illustrates the
proportionality of each measurement distribution conditioned by the state
$p(\mathbf{z}^{[i]}|\mathbf{x})$ over the corresponding residual error
$\mathbf{e}^{[i]}(\mathbf{x})$
defined in \eqref{eq:distance-maps:background:least-squares:error},
where $\mathbf{\Omega}^{[i]}$ is the information matrix,
$\mathbf{h}^{[i]}(\mathbf{x})$ represents the predicted measurement given the
state estimate $\mathbf{x}$, and $\mathbf{z}^{[i]}$ the measurement itself.
\begin{equation}\label{eq:distance-maps:background:least-squares:independence}
p(\mathbf{x}|\mathbf{z}) \propto \prod_i p(\mathbf{z}^{[i]}|\mathbf{x})
\end{equation}
\begin{equation}\label{eq:distance-maps:background:least-squares:measurement}
\begin{aligned}
p(\mathbf{z}^{[i]}|\mathbf{x})
  & = \mathcal{N}(\mathbf{z}^{[i]} ; \mathbf{h}^{[i]}(\mathbf{x}), \mathbf{\Sigma^{[i]}})\\
  & \propto \exp\left(
      -\frac{1}{2}
      \left( \mathbf{h}^{[i]}(\mathbf{x}) - \mathbf{z}^{[i]} \right)^T
      \left(\mathbf{\Sigma^{[i]}}\right)^{-1}
      \left( \mathbf{h}^{[i]}(\mathbf{x}) - \mathbf{z}^{[i]} \right)
  \right)\\
  & \propto \exp\left(
      -\frac{1}{2}
      \mathbf{e}^{[i]}(\mathbf{x})^T
      \mathbf{\Omega^{[i]}}
      \mathbf{e}^{[i]}(\mathbf{x})
  \right)
\end{aligned}
\end{equation}
\begin{equation}\label{eq:distance-maps:background:least-squares:error}
\mathbf{e}^{[i]}(\mathbf{x})=\mathbf{h}^{[i]}(\mathbf{x}) - \mathbf{z}^{[i]}
\end{equation}

As a result, under the assumption of having an uniform prior,
the original \gls{map} problem formulated in
\eqref{eq:distance-maps:background:least-squares:map} can be rewritten as
an \gls{mle} problem that maximizes the probability of the measurements given
the state $p(\mathbf{z}^{[i]}|\mathbf{x})$
(see \eqref{eq:distance-maps:background:least-squares:mle-log-likelihood}).
Thus, through the log-likelihood over the conditioned measurement distribution
(see \defref{def:literature:slam:log-likelihood}) and assuming the measurements
under Gaussian noise, the state estimation in robotics from
the \gls{mle} formulation can be formulated as a non-linear least-squares
problem, where the objective is to minimize the sum of squared residuals
weighted by the corresponding information matrices.
\begin{equation}\label{eq:distance-maps:background:least-squares:mle-log-likelihood}
\begin{aligned}
\mathbf{x}^*
  & = \argmax_\mathbf{x} \prod_i p(\mathbf{z}^{[i]}|\mathbf{x})\\
  & = \argmax_\mathbf{x} \prod_i \exp\left( -\frac{1}{2}\mathbf{e}^{[i]}(\mathbf{x})^T \mathbf{\Omega}^{[i]} \mathbf{e}^{[i]}(\mathbf{x}) \right)\\
  & = \argmin_\mathbf{x} \sum_i \mathbf{e}^{[i]}(\mathbf{x})^T \mathbf{\Omega}^{[i]} \mathbf{e}^{[i]}(\mathbf{x})
\end{aligned}
\end{equation}

\subsubsection{Gauss-Newton algorithm}\label{sec:distance-maps:background:least-squares:gn}

Following the non-linear least-squares formulation commonly used in
robotics state estimation, one possible solution is an iterative minimization of
\eqref{eq:distance-maps:background:least-squares:iterative-function}~\cite{grisetti:robotics:2020}.
At each iteration $k$, the current estimate $\mathbf{x}_k$ is updated by a
perturbation $\mathbf{\Delta x}_k$, as shown in
\eqref{eq:distance-maps:background:least-squares:perturbation}.
\begin{equation}\label{eq:distance-maps:background:least-squares:iterative-function}
F\left(\mathbf{x}_k\right) =
\sum_i \mathbf{e}^{\left[i\right]}\left(\mathbf{x}_k\right)^T
       \mathbf{\Omega}^{\left[i\right]}
       \mathbf{e}^{\left[i\right]}\left(\mathbf{x}_k\right)
\end{equation}
\begin{equation}\label{eq:distance-maps:background:least-squares:perturbation}
\mathbf{x}_{k+1} = \mathbf{x}_k + \mathbf{\Delta x}_k
\end{equation}

In order to compute the state perturbation $\mathbf{\Delta x}_k$,
we can linearize each error function around the current estimate $\mathbf{x}_k$
using a first-order Taylor approximation, as formulated in
\eqref{eq:distance-maps:background:least-squares:error-function-taylor}.
\begin{equation}\label{eq:distance-maps:background:least-squares:error-function-taylor}
\mathbf{e}^{\left[i\right]}\left(\mathbf{x}_k + \mathbf{\Delta x}_k\right)
\simeq
\underbrace{\mathbf{e}^{\left[i\right]}\left(\mathbf{x}_k\right)}_{\mathbf{e}^{\left[i\right]}} +
\underbrace{\left.\frac{\partial \mathbf{e}^{\left[i\right]}\left(\mathbf{x}\right)}{\partial \mathbf{x}}
\right|_{\mathbf{x}=\mathbf{x}_k}}_{\mathbf{J}^{\left[i\right]}}
\mathbf{\Delta x}_k
\end{equation}

\eqref{eq:distance-maps:background:least-squares:products-expansion}
expands the products of the first-order Taylor approximation in
\eqref{eq:distance-maps:background:least-squares:iterative-function},
considering the state perturbation update shown in
\eqref{eq:distance-maps:background:least-squares:perturbation}.
This expansion considers the state variables
$\mathbf{x}\in\mathbb{R}^n$,
the error function $\mathbf{e}^{\left[i\right]}\in\mathbb{R}^m$,
the information matrix $\mathbf{\Omega}^{\left[i\right]}\in\mathbb{R}^{m\times m}$,
and the Jacobian matrix $\mathbf{J}^{\left[i\right]}\in\mathbb{R}^{m\times n}$.
Since each term in the expression from
\eqref{eq:distance-maps:background:least-squares:products-expansion} is a
scalar, $\mathbf{\Delta x}^T {\mathbf{J}^{\left[i\right]}}^T
\mathbf{\Omega}^{\left[i\right]} \mathbf{e}^{\left[i\right]} =
{\mathbf{e}^{\left[i\right]}}^T \mathbf{\Omega}^{\left[i\right]}
\mathbf{J}^{\left[i\right]} \mathbf{\Delta x}$.
\begin{equation}\label{eq:distance-maps:background:least-squares:products-expansion}
\begin{aligned}
\mathbf{e}^{\left[i\right]}\left(\mathbf{x}_k + \mathbf{\Delta x}_k\right)^T
\mathbf{\Omega}^{\left[i\right]}
\mathbf{e}^{\left[i\right]}\left(\mathbf{x}_k + \mathbf{\Delta x}_k\right)
& \simeq & \left(\mathbf{e}^{\left[i\right]} + \mathbf{J}^{\left[i\right]} \mathbf{\Delta x}\right)^T
         \mathbf{\Omega}^{\left[i\right]}
         \left(\mathbf{e}^{\left[i\right]} + \mathbf{J}^{\left[i\right]} \mathbf{\Delta x}\right) & =\\
& = & {\mathbf{e}^{\left[i\right]}}^T \mathbf{\Omega}^{\left[i\right]} \mathbf{e}^{\left[i\right]} +
      {\mathbf{e}^{\left[i\right]}}^T \mathbf{\Omega}^{\left[i\right]} \mathbf{J}^{\left[i\right]} \mathbf{\Delta x} & \; +\\
&   & \mathbf{\Delta x}^T {\mathbf{J}^{\left[i\right]}}^T \mathbf{\Omega}^{\left[i\right]} \mathbf{e}^{\left[i\right]} +
      \mathbf{\Delta x}^T {\mathbf{J}^{\left[i\right]}}^T \mathbf{\Omega}^{\left[i\right]} \mathbf{J}^{\left[i\right]} \mathbf{\Delta x} & =\\
& = & \mathbf{\Delta x}^T \underbrace{{\mathbf{J}^{\left[i\right]}}^T \mathbf{\Omega}^{\left[i\right]} \mathbf{J}^{\left[i\right]}}_{\mathbf{H}^{\left[i\right]}} \mathbf{\Delta x} +
      2 \underbrace{{\mathbf{e}^{\left[i\right]}}^T \mathbf{\Omega}^{\left[i\right]} \mathbf{J}^{\left[i\right]}}_{{\mathbf{b}^{\left[i\right]}}^T} \mathbf{\Delta x} +
      \underbrace{{\mathbf{e}^{\left[i\right]}}^T \mathbf{\Omega}^{\left[i\right]} \mathbf{e}^{\left[i\right]}}_{c^{\left[i\right]}} &
\end{aligned}
\end{equation}

Next,
\eqref{eq:distance-maps:background:least-squares:iterative-function-linearized}
groups the terms illustrated in
\eqref{eq:distance-maps:background:least-squares:products-expansion}
to formulate a quadratic form in the perturbation of the state variables,
considering $\mathbf{H} = \sum_i \mathbf{H}^{\left[i\right]}\in\mathbb{R}^{n\times n}$,
$\mathbf{b} = \sum_i \mathbf{b}^{\left[i\right]}\in\mathbb{R}^{n}$, and
$c = \sum_i c^{\left[i\right]}\in\mathbb{R}$.
This quadratic form linearizes the original non-linear least-squares problem
formulated in
\eqref{eq:distance-maps:background:least-squares:mle-log-likelihood}
at each iteration of the minimization, considering the perturbation update of
the state variables in
\eqref{eq:distance-maps:background:least-squares:perturbation}.
\begin{equation}\label{eq:distance-maps:background:least-squares:iterative-function-linearized}
F\left(\mathbf{x}_k + \mathbf{\Delta x}_k\right)
\simeq
\mathbf{\Delta x}_k^T \mathbf{H} \mathbf{\Delta x}_k +
2 \mathbf{b}^T \mathbf{\Delta x} +
c
\end{equation}

The perturbation $\mathbf{\Delta x}_k$ is obtained by minimizing the quadratic
approximation of the objective function (see
\eqref{eq:distance-maps:background:least-squares:perturbation-minimization}).
This minimization is done by setting the gradient of the quadratic approximation
\gls{wrt} $\mathbf{\Delta x}_k$ to zero, as shown in
\eqref{eq:distance-maps:background:least-squares:perturbation-minimization-solution}.
\begin{equation}\label{eq:distance-maps:background:least-squares:perturbation-minimization}
\begin{aligned}
\mathbf{\Delta x}_k^*
& = \argmin_{\mathbf{\Delta x}}
    F\left(\mathbf{x}_k + \mathbf{\Delta x}_k\right)\\
& \simeq \argmin_{\mathbf{\Delta x}}
    \mathbf{\Delta x}_k^T \mathbf{H} \mathbf{\Delta x}_k +
    2 \mathbf{b}^T \mathbf{\Delta x} + c
\end{aligned}
\end{equation}
\begin{equation}\label{eq:distance-maps:background:least-squares:perturbation-minimization-solution}
\begin{aligned}
\mathbf{0}
& = \frac{\partial \left[ \mathbf{\Delta x}_k^T \mathbf{H} \mathbf{\Delta x}_k +
    2 \mathbf{b}^T \mathbf{\Delta x} + c \right]}{\partial \mathbf{\Delta x}}\\
\mathbf{0}
& = \left( \mathbf{H} + \mathbf{H}^T \right) \mathbf{\Delta x}_k + 2 \mathbf{b}\\
\mathbf{0}
& = 2 \mathbf{H} \mathbf{\Delta x}_k + 2 \mathbf{b}\\
-\mathbf{b} & = \mathbf{H} \mathbf{\Delta x}_k
\end{aligned}
\end{equation}

Therefore, at iteration $k$, the perturbation $\mathbf{\Delta x}_k$ is obtained
by solving the linear system presented in
\eqref{eq:distance-maps:background:least-squares:perturbation-minimization-solution}.
The estimate is then updated following
\eqref{eq:distance-maps:background:least-squares:perturbation}.
This iterative procedure corresponds to the Gauss-Newton method, which
linearizes the residual error functions around the current estimate and
minimizes the resulting quadratic approximation of the weighted non-linear
least-squares cost~\cite{grisetti:robotics:2020}.
\algref{alg:distance-maps:background:least-squares:gauss-newton} details the
steps of the Gauss-Newton minimization algorithm for a single iteration of the
iterative procedure.
\begin{equation}\label{eq:distance-maps:background:least-squares:b-hx}
\mathbf{H} \mathbf{\Delta x}_k = -\mathbf{b}
\end{equation}

\begin{algorithm}[!t]
\DontPrintSemicolon
\caption[Gauss-Newton minimization algorithm for Euclidean measurement and state
spaces]{Gauss-Newton minimization algorithm for Euclidean measurement and
state spaces (one single iteration of the iterative procedure)}
\label{alg:distance-maps:background:least-squares:gauss-newton}
\KwIn{Current state estimation $\mathbf{x}_k$,
measurements $\mathcal{C} = {\langle \mathbf{z}_i , \mathbf{\Omega}_i \rangle}$}
\KwOut{State estimation for the next iteration step $\mathbf{x}_{k+1}$}
$\mathbf{b} \leftarrow \mathbf{0}_{n \times 1}$\;
$\mathbf{H} \leftarrow \mathbf{0}_{n \times n}$\;
\ForAll{$i$}{
  $\mathbf{e}^{[i]} \leftarrow \mathbf{h}^{[i]} - \mathbf{z}^{[i]}$\;
  $\mathbf{J}^{\left[i\right]} \leftarrow \left.\frac{\partial
      \mathbf{e}^{\left[i\right]}\left(\mathbf{x}\right)}{\partial \mathbf{x}}
      \right|_{\mathbf{x}=\mathbf{x}_k}$\;
  $\mathbf{H} \leftarrow \mathbf{H} + {\mathbf{J}^{\left[i\right]}}^T
      \mathbf{\Omega}_i \mathbf{J}^{\left[i\right]}$\;
  $\mathbf{b} \leftarrow \mathbf{b} + {\mathbf{J}^{\left[i\right]}}^T
      \mathbf{\Omega}_i \mathbf{e}^{\left[i\right]}$\;
}
$\mathbf{\Delta x}_k \leftarrow \text{solve}(\mathbf{H} \mathbf{\Delta x}_k = -\mathbf{b})$\;
$\mathbf{x}_{k+1} \leftarrow \mathbf{x}_{k} + \mathbf{\Delta x}_k$\;
\Return{$\mathbf{x}_{k+1}$}\;
\end{algorithm}
